Who does what

DAEMON (adlrd)
  • Single writer to SQLite
  • Spawns & owns agent child processes
  • Streams agent stdout → SQLite log
  • Pushes events to connected clients
  • Auto-starts on first CLI use
  • Persists beyond terminal sessions
CLI (adlr)
  • Thin client — sends commands to daemon
  • Receives responses, prints to stdout
  • Auto-starts daemon if not running
TUI (adlr with no args)
  • Holds open socket connection to daemon
  • Receives push events, re-renders on each
  • Built with Ink (React for terminal)

IPC Protocol (Unix socket, newline-delimited JSON)

// CLI → daemon (command)
{"type":"agent.run","id":"req-1","session":"abc","payload":{"prompt":"...","agent":"opencode:build"}}
// daemon → CLI (response)
{"type":"response","id":"req-1","payload":{"agentId":"agent-xyz","status":"running"}}
// daemon → TUI (event stream)
{"type":"event","event":"agent.output","payload":{"agentId":"agent-xyz","line":"Building..."}}
Persistence abstraction: The daemon wraps all SQLite access behind a Storage interface. Swapping the backend later only requires re-implementing that interface.
