adlr — Daemon & CLI Design

Section 3 of 4 — Package structure, daemon lifecycle, and IPC flows

Monorepo packages

packages/
core/     — shared types, Storage interface, IPC protocol
daemon/   — adlrd process
cli/      — adlr CLI binary
tui/      — Ink dashboard (invoked by CLI)
opencode/  — @adlr/opencode plugin

Daemon lifecycle

start: auto-spawned by CLI if socket missing
→ detached Bun process, writes PID to ~/.local/share/adlr/
socket: ~/.local/share/adlr/adlr.sock
→ one global daemon, all sessions managed here
stop: SIGTERM, or after 10min with no clients
→ graceful: wait for running agents to finish
db: ~/.local/share/adlr/adlr.db

IPC flows (newline-delimited JSON)

CLI command flow
CLI  →  connect to socket
CLI  →  {"type":"agent.run","id":"r1","payload":{...}}
daemon  →  {"type":"response","id":"r1","payload":{...}}
CLI  →  print result, close connection
TUI subscription flow
TUI  →  {"type":"subscribe","payload":{"session_id":"..."}}
daemon  →  {"type":"snapshot","payload":{spans,events,context}}
daemon  →  {"type":"event","event":"span.started",...}
daemon  →  {"type":"event","event":"event.created",...}
← connection stays open, daemon pushes on every change
Agent spawn flow
daemon creates span (kind=agent) in Storage
daemon Bun.spawn(agentCmd, {
env: { ADLR_SESSION, ADLR_SPAN_ID,
ADLR_SOCKET, ADLR_CONTEXT, ... }
})
daemon streams stdout → event(agent.output) → push to TUI
on exit → span.status = done/failed, push to TUI

CLI commands

adlr
open TUI dashboard for current session
adlr new [--goal "..."]
create session, optionally set goal context item
adlr agent run
command daemon to spawn agent, returns span id
adlr agent wait/status/list
query span state via daemon
adlr context add/list/get
manage context items for current session
adlr session list
list all sessions across all projects
adlr daemon stop
graceful shutdown
