What should we spec today?

Each phase builds on the previous. Phase 1 is required for everything else.

A

Phase 1 — Core Foundation

Sessions, CLI, Agent Runner, Context system. The bedrock everything else builds on. No UI yet — just the data model and command-line interface.

Deliverables: adlr new, adlr agent run/wait/status, adlr context add/list, session persistence

B

Full Concept Architecture

Design the whole system top-to-bottom first — data model, component boundaries, tech stack decisions — then produce phase-by-phase implementation plans.

Deliverables: High-level architecture spec covering all 8 subsystems, then per-phase plans

C

TUI-First (Phase 1 + 2 together)

Core foundation plus the TUI dashboard in one spec. Useful if the TUI shapes fundamental design decisions about how session data is structured and observed.

Deliverables: Everything in Phase 1, plus live TUI dashboard
